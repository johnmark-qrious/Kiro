# Requirements Document

## Introduction

This feature adds a "No File Notification Schedule" UI to the connector setup wizard's Notifications step. When a user enables the no-file notification toggle, a schedule builder is revealed allowing them to define when the system should check for missing files and alert via notification. The schedule supports four frequency types (Hourly, Daily, Weekly, Monthly) each with frequency-specific field sets. User selections are serialised into a `file_reminder_config` payload containing a valid RRule string (with `DTSTART`, `TZID=Pacific/Auckland`, and one of `FREQ=HOURLY`, `FREQ=DAILY`, `FREQ=WEEKLY`, or `FREQ=MONTHLY`) and persisted alongside the connector configuration via the existing connector creation gRPC flow. No new backend endpoint is required — the existing `createConnectorVariable` flow handles it.

## Glossary

- **Connector_Wizard**: The multi-step form (`AddConnectorWizard`) used to create a new importer or extractor connector. Steps include Setup, Connection, Notifications, Configuration, Mapping, and Review.
- **Notifications_Step**: The wizard step (`NotificationsStep`) where users configure success/failure email recipients and, with this feature, the no-file notification schedule.
- **Schedule_Builder**: The UI component that renders frequency-specific fields (interval dropdowns, time pickers, day-of-week toggles, day-of-month chips, ordinal selectors) and produces an RRule string.
- **Frequency_Selector**: A split-button (radio toggle group) allowing the user to choose between Hourly, Daily, Weekly, and Monthly frequency types.
- **RRule_String**: A recurrence rule string conforming to the iCalendar RFC 5545 RRULE specification, generated by the `rrule` library. Must include `DTSTART` with `TZID=Pacific/Auckland` and use one of the four backend-allowed frequencies: `FREQ=HOURLY`, `FREQ=DAILY`, `FREQ=WEEKLY`, or `FREQ=MONTHLY`. `FREQ=MINUTELY` is explicitly rejected by backend validation.
- **File_Reminder_Config**: The backend payload shape `{ file_check_type: string, file_check_schedule: string }` defined by `FileReminderCheckConfigSchema` in `@monorepo/packages-schemas`. The `file_check_schedule` field contains the RRule string. The backend validates the RRULE frequency is one of HOURLY, DAILY, WEEKLY, or MONTHLY.
- **No_File_Toggle**: A toggle switch on the Notifications step that controls visibility of the Schedule_Builder for no-file notifications.
- **Day_Of_Month_Chips**: A multi-select input rendered as removable chip elements, allowing the user to select one or more days of the month (1–31).
- **Ordinal_Selector**: A dropdown allowing selection of positional values (first, second, third, fourth, last) used in the Monthly Day pattern.
- **Connector_Form**: The TanStack React Form instance (`useAddConnectorForm`) that manages all wizard state including the new `fileReminderConfig` field.

## Prerequisites and Open Questions

### Known Integration Points
1. The `FileReminderCheckConfigSchema` in `monorepo/packages/schemas/src/prefect-connector.schemas.ts` defines the payload as `{ file_check_type: string, file_check_schedule: string }`. The backend validates the RRULE frequency is one of `FREQ=HOURLY`, `FREQ=DAILY`, `FREQ=WEEKLY`, or `FREQ=MONTHLY`. `FREQ=MINUTELY` is explicitly rejected.
2. The `connectorConfigTransformer.ts` currently hardcodes `file_reminder_config: null` for all connector types. This must be updated to pass through the user's schedule configuration.
3. The existing RRule schedule builder (`rrule-generator/`) provides reusable patterns for Daily, Weekly, and Monthly frequencies but does not support Hourly. The no-file schedule builder requires Hourly support and different interval ranges than the existing connector schedule builder. A new builder should be created following the same Context+Reducer pattern.
4. The `ConnectorFormData` type must be extended with a `fileReminderConfig` field.
5. No new backend endpoint is required. The existing `createConnectorVariable` flow accepts `file_reminder_config` in the `ConfigCreationRequest` payload.
6. No-file notifications reuse the existing `failure_recipients` — no separate recipient list is needed.

### Resolved Questions
1. **~~Hourly sub-hour intervals~~**: Resolved — sub-hour intervals (0.25, 0.5) are not needed. The smallest unit is 1 hour. Backend rejects `FREQ=MINUTELY`.
2. **~~file_check_type mapping~~**: Resolved — there is no separate "check type" concept. The backend accepts a raw RRULE string with a FREQ value. The `file_check_type` field in the schema still needs a value, but the RRULE frequency IS the schedule.
3. **~~Timezone handling~~**: Resolved — use hardcoded `"Pacific/Auckland"` consistent with the existing schedule builder. RRULE must include `TZID=Pacific/Auckland`.
4. **~~No-file notification recipients~~**: Resolved — backend uses existing `failure_recipients`. No UI change needed.
5. **~~Backend endpoint~~**: Resolved — no new endpoint needed. Existing `createConnectorVariable` flow handles it.

## Requirements

### Requirement 1: No-File Notification Toggle

**User Story:** As a connector administrator, I want to enable or disable no-file notifications via a toggle switch on the Notifications step, so that I can opt in to missing-file alerts without affecting other notification settings.

#### Acceptance Criteria

1. THE Notifications_Step SHALL display a No_File_Toggle switch labelled "No File Notification" below the existing email notification fields.
2. WHEN the No_File_Toggle is switched on, THE Notifications_Step SHALL reveal the Schedule_Builder section with a smooth transition.
3. WHEN the No_File_Toggle is switched off, THE Notifications_Step SHALL hide the Schedule_Builder section and clear all schedule-related form state.
4. THE No_File_Toggle SHALL default to the off position when the Notifications step is first rendered.

### Requirement 2: Start Date and Time Picker

**User Story:** As a connector administrator, I want to set a start date and time for the no-file notification schedule, so that monitoring begins at a specific point in time.

#### Acceptance Criteria

1. WHILE the No_File_Toggle is on, THE Schedule_Builder SHALL display a start date picker and a start time picker.
2. THE Schedule_Builder SHALL pre-fill the start date picker with the current date.
3. THE Schedule_Builder SHALL pre-fill the start time picker with the current time rounded to the nearest hour.
4. THE Schedule_Builder SHALL prevent selection of a start date in the past.
5. THE start date picker and start time picker SHALL be required fields for all frequency types.

### Requirement 3: Frequency Selection

**User Story:** As a connector administrator, I want to choose between Hourly, Daily, Weekly, and Monthly frequencies, so that I can match the notification schedule to the expected file delivery cadence.

#### Acceptance Criteria

1. WHILE the No_File_Toggle is on, THE Schedule_Builder SHALL display a Frequency_Selector with four options: Hourly, Daily, Weekly, and Monthly.
2. THE Frequency_Selector SHALL render as a radio toggle group (split button) with mutually exclusive selection.
3. WHEN a frequency option is selected, THE Schedule_Builder SHALL display only the field set corresponding to the selected frequency.
4. WHEN a new frequency option is selected, THE Schedule_Builder SHALL reset all frequency-specific fields to their default values.
5. THE Frequency_Selector SHALL default to Daily when the Schedule_Builder is first rendered.

### Requirement 4: Hourly Frequency Configuration

**User Story:** As a connector administrator, I want to configure an hourly no-file check interval, so that I receive alerts within hours of a missed file delivery.

#### Acceptance Criteria

1. WHEN Hourly frequency is selected, THE Schedule_Builder SHALL display an "Every" dropdown.
2. THE "Every" dropdown for Hourly frequency SHALL contain integer values 1 through 23.
3. THE "Every" dropdown for Hourly frequency SHALL display values with "hour(s)" suffix label.
3. THE "Every" dropdown for Hourly frequency SHALL default to 1.
4. WHEN Hourly frequency is selected, THE Schedule_Builder SHALL hide the "At" time picker, day-of-week toggles, day-of-month chips, and ordinal selectors.

### Requirement 5: Daily Frequency Configuration

**User Story:** As a connector administrator, I want to configure a daily no-file check schedule, so that I receive alerts on a daily cadence at a specific time.

#### Acceptance Criteria

1. WHEN Daily frequency is selected, THE Schedule_Builder SHALL display an "Every" dropdown and an "At" time picker.
2. THE "Every" dropdown for Daily frequency SHALL contain values 1 through 6.
3. THE "Every" dropdown for Daily frequency SHALL default to 1.
4. THE "At" time picker for Daily frequency SHALL default to the start time value.

### Requirement 6: Weekly Frequency Configuration

**User Story:** As a connector administrator, I want to configure a weekly no-file check schedule with specific days, so that I receive alerts aligned with weekly file delivery expectations.

#### Acceptance Criteria

1. WHEN Weekly frequency is selected, THE Schedule_Builder SHALL display an "Every" dropdown, an "On" day-of-week toggle group, and an "At" time picker.
2. THE "Every" dropdown for Weekly frequency SHALL contain values 1 through 10.
3. THE "Every" dropdown for Weekly frequency SHALL default to 1.
4. THE "On" day-of-week toggle group SHALL render seven toggle buttons (Monday through Sunday) supporting multi-select.
5. THE "On" day-of-week toggle group SHALL default to Monday selected.
6. THE "On" day-of-week toggle group SHALL prevent deselecting the last remaining selected day.
7. THE "At" time picker for Weekly frequency SHALL default to the start time value.

### Requirement 7: Monthly Frequency Configuration — Day Pattern

**User Story:** As a connector administrator, I want to configure a monthly no-file check on a specific ordinal day (e.g., "first Monday"), so that I can align alerts with monthly delivery patterns.

#### Acceptance Criteria

1. WHEN Monthly frequency is selected, THE Schedule_Builder SHALL display a "Pattern" toggle with "Day" and "Date" options.
2. WHEN the "Day" pattern is selected, THE Schedule_Builder SHALL display an "Every" dropdown, an Ordinal_Selector, a day-of-week selector, and an "At" time picker.
3. THE "Every" dropdown for Monthly Day pattern SHALL contain values 1 through 12.
4. THE "Every" dropdown for Monthly Day pattern SHALL default to 1.
5. THE Ordinal_Selector SHALL contain the options: first, second, third, fourth, last.
6. THE day-of-week selector for Monthly Day pattern SHALL contain: Monday, Tuesday, Wednesday, Thursday, Friday, Saturday, Sunday, weekday, weekend day.
7. THE Ordinal_Selector SHALL default to "first".
8. THE day-of-week selector SHALL default to "Monday".
9. THE "At" time picker for Monthly Day pattern SHALL default to the start time value.

### Requirement 8: Monthly Frequency Configuration — Date Pattern

**User Story:** As a connector administrator, I want to configure a monthly no-file check on specific calendar dates, so that I can align alerts with date-based delivery schedules.

#### Acceptance Criteria

1. WHEN the "Date" pattern is selected, THE Schedule_Builder SHALL display an "Every" dropdown, a Day_Of_Month_Chips input, and an "At" time picker.
2. THE "Every" dropdown for Monthly Date pattern SHALL contain values 1 through 12.
3. THE "Every" dropdown for Monthly Date pattern SHALL default to 1.
4. THE Day_Of_Month_Chips input SHALL allow selection of one or more days from 1 to 31.
5. THE Day_Of_Month_Chips input SHALL render selected days as removable chip elements.
6. THE Day_Of_Month_Chips input SHALL allow individual removal of selected day chips.
7. THE Day_Of_Month_Chips input SHALL prevent removal of the last remaining selected day.
8. THE Day_Of_Month_Chips input SHALL default to day 1 selected.
9. THE "At" time picker for Monthly Date pattern SHALL default to the start time value.
10. THE "Pattern" toggle SHALL default to "Day" when Monthly frequency is first selected.


### Requirement 9: Form Validation

**User Story:** As a connector administrator, I want the system to validate my schedule configuration before I proceed, so that I do not submit an incomplete or invalid schedule.

#### Acceptance Criteria

1. WHILE the No_File_Toggle is on, THE Connector_Wizard SHALL prevent progression to the next step until all required schedule fields for the selected frequency are complete.
2. WHEN the user attempts to progress with missing required fields, THE Schedule_Builder SHALL display inline validation error messages adjacent to each incomplete field.
3. WHILE the No_File_Toggle is off, THE Connector_Wizard SHALL allow progression without schedule validation.
4. THE validation SHALL check that at least one day is selected for Weekly frequency day-of-week toggles.
5. THE validation SHALL check that at least one day is selected for Monthly Date pattern Day_Of_Month_Chips.
6. THE validation SHALL check that the start date is not in the past at the time of validation.
7. IF the user corrects a validation error, THEN THE Schedule_Builder SHALL remove the corresponding inline error message immediately.

### Requirement 10: Schedule Payload Serialisation

**User Story:** As a connector administrator, I want my schedule selections to be serialised into the correct backend payload format, so that the backend can generate and persist the notification schedule.

#### Acceptance Criteria

1. THE Connector_Form SHALL store the no-file schedule as a `fileReminderConfig` field containing `file_check_type` and `file_check_schedule` properties.
2. THE Schedule_Builder SHALL generate a valid RRule_String using only `FREQ=HOURLY`, `FREQ=DAILY`, `FREQ=WEEKLY`, or `FREQ=MONTHLY` frequencies, with `DTSTART` and `TZID=Pacific/Auckland`, and store it as the `file_check_schedule` value.
3. THE Schedule_Builder SHALL NOT generate RRule strings with `FREQ=MINUTELY` or any frequency other than the four allowed by the backend.
4. THE `connectorConfigTransformer` SHALL include the `fileReminderConfig` value in the `file_reminder_config` field of the connector creation payload when the No_File_Toggle is on.
5. WHEN the No_File_Toggle is off, THE `connectorConfigTransformer` SHALL set `file_reminder_config` to null in the connector creation payload.
6. FOR ALL valid schedule configurations, serialising the configuration to an RRule_String and parsing the RRule_String back SHALL produce an equivalent schedule configuration (round-trip property).

### Requirement 11: Frequency Change Reset Behaviour

**User Story:** As a connector administrator, I want all frequency-specific fields to reset when I switch frequencies, so that stale values from a previous frequency do not carry over.

#### Acceptance Criteria

1. WHEN the user switches from one frequency to another, THE Schedule_Builder SHALL reset the interval dropdown to the new frequency's default value.
2. WHEN the user switches from one frequency to another, THE Schedule_Builder SHALL clear all day-of-week selections and reset to the new frequency's defaults.
3. WHEN the user switches from one frequency to another, THE Schedule_Builder SHALL clear all day-of-month selections and reset to the new frequency's defaults.
4. WHEN the user switches from one frequency to another, THE Schedule_Builder SHALL clear the ordinal and day-of-week selectors for Monthly Day pattern.
5. WHEN the user switches from one frequency to another, THE Schedule_Builder SHALL retain the start date and start time values.
6. WHEN the user switches from one frequency to another, THE Schedule_Builder SHALL clear any existing validation errors.

### Requirement 12: Review Step Display

**User Story:** As a connector administrator, I want to see a summary of my no-file notification schedule on the Review step, so that I can verify the configuration before creating the connector.

#### Acceptance Criteria

1. WHILE the No_File_Toggle is on, THE Review step SHALL display a "No File Notification Schedule" section showing a human-readable summary of the configured schedule.
2. WHILE the No_File_Toggle is off, THE Review step SHALL not display the "No File Notification Schedule" section.
3. THE human-readable summary SHALL include the frequency type, interval, selected days (if applicable), and scheduled time (if applicable).
