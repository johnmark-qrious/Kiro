# Design Review Checklist

## Problem Definition
- Is this solving a real user problem?
- Do we have evidence users need this?
- What's the simplest solution?

## Complexity
- Is this over-engineered?
- Are we solving hypothetical problems?
- Can we use existing patterns/code?
- What's the maintenance burden?

## Scope
- Can we ship a smaller version first?
- What's the MVP?
- What can we defer or cut?

## Questions to Ask

- "Do users actually need this, or is it nice-to-have?"
- "Can we ship without this and add it later if needed?"
- "What's the simplest version that solves the actual problem?"
- "Why do we need this new abstraction? Can we just use the existing pattern?"
- "This adds complexity - what problem does it solve that we can't solve more simply?"
